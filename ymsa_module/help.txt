(?) 帮助文档

(i) 说明
括号里的数字是警报代码，在警报弹窗里也有，您可以通过该代码快速寻找到文档中对应的段落
该文档只能尽力列出常见问题的原因和可能的解决方法
对于该文档没能预料到的问题，您应在必要时询问AI

(1) 导入Json文件失败
=> 等级：红（脚本故障）
=> 触发条件
尝试读取「.\ymsa_module\user_config.json」时找不到该文件
=> 可能的原因
该文件不存在
=> 可能的解决方法
补上该文件

(2) 解析Json文件失败
=> 等级：红（脚本故障）
=> 触发条件
尝试解析「.\ymsa_module\user_config.json」的内容时出错
=> 可能的原因
括号不成对、引号不成对
中间漏逗号、末尾多逗号
使用了全角标点符号
路径分隔符没有使用两个反斜杠「C:\\Users\\Yachiyo\\...」
在Json里写注释「// 就像这样」
=> 可能的解决方法
Json有着极其严苛的语法要求，建议优先把配置文件丢给AI询问

(3) Java路径为空
=> 等级：红（脚本故障）
=> 触发条件
「.\ymsa_module\user_config.json」的「javaPath」对象为空
=> 可能的原因
没写
=> 可能的解决方法
补上

(4) Java参数类型必须是数组
=> 等级：红（脚本故障）
=> 触发条件
「.\ymsa_module\user_config.json」的「javaArgs」对象类型不是数组
=> 可能的原因
把数组「["param1","param2"]」写成了字符串「"param1 param2"」
=> 可能的解决方法
改

(5) Java参数为空
=> 等级：红（脚本故障）
=> 触发条件
「.\ymsa_module\user_config.json」的「javaArgs」对象是空数组
=> 可能的原因
没写
=> 可能的解决方法
补上

(6) Java路径无效
=> 等级：红（脚本故障）
=> 触发条件
「.\ymsa_module\user_config.json」的「javaPath」对象的值无效
=> 可能的原因
路径末尾没精确到或不是「java.exe」
路径写错了
=> 可能的解决方法
改

(7) 找不到Java路径指定的文件
=> 等级：红（脚本故障）
=> 触发条件
「.\ymsa_module\user_config.json」的「javaPath」对象里的路径在设备上不存在
=> 可能的原因
路径写错了
=> 可能的解决方法
仔细检查，改

(8) 因意外退出进入安全模式
=> 等级：橙（服务端故障）
=> 触发条件
启动脚本时，检测到「.\ymsa_module\temp_running_flag」文件存在
=> 可能的原因
服务端与系统冲突，导致系统死机、蓝屏等
设备意外断电
直接按「×」关掉了脚本
脚本进程意外被杀
=> 可能的解决方法
如果设备刚刚死机、蓝屏等，不要再尝试开服，备份好重要数据，确认问题已解决后再继续
如果设备刚刚意外断电，在确认无其它问题后可以继续
正常情况不应按「×」关服
删除「.\ymsa_module\temp_running_flag」后解除锁定
